Parameters for various operations for the front end design integration of rabbit mq listed here:

Parameters for sending and receiving messages to the queue(s): 

operation, ip_address, port_number, username, password, vhost_name, queue_name


1. Authentication: 
	
	A) Sign-In (Producer):
		operation: 'login'
		username: <username>
		password: <password>

	B) Sign-In (Consumer):
		operation: 'login'
		result: 'true or false'

	C) Register (Producer):
		operation: 'register'
		username: <username>
		password: <password>
		email: <email>

	D) Register (Consumer):
		operation: 'register'
		result: 'true or false'

2. Youtube API query:
	
	A) Search (Producer):
		operation: 'youtube-search'
		search: <search_terms>
		api-key: <api_key>

	B) Search (Consumer):
		operation: 'youtuber-search'
		payload: <json_payload>

	C) API request parts:
		root-url: 'https://www.googleapis.com/youtube/v3/search?'
		part: part=snippet
		q: q=<search_terms>
		key: key=<API_KEY>

	D)  Required API Request Info to Return for Search Results:
		video-id: <video_id>
		title: <title>
		thumbnail: <thumbnail_url>
		
		*Only send first 25 results!
